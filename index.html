What is Claude Code?

An AI-powered coding assistant in your terminal

  • Command-line interface (CLI) tool by Anthropic
  • Helps with coding tasks, debugging, and project management
  • Understands your entire codebase context
  • Can read, write, and modify files automatically
  • Integrates with your existing development workflow

Installation & Setup

Get up and running in under 2 minutes

1. Install Claude Code

# Option 1: Using npm
npm install -g @anthropics/claude-code

# Option 2: Download from Anthropic
# Visit: https://claude.ai/code

2. Authenticate

claude auth login

3. Start Coding!

# Navigate to your project
cd your-project

# Start Claude Code
claude

What is CLAUDE.md?

Think of it as instructions for your AI assistant

  • A special file you create in your project folder
  • Contains project-specific information for Claude
  • Like giving Claude a "user manual" for your project
  • Helps Claude understand your project's unique setup

CLAUDE.md Example

Here's what a real CLAUDE.md file looks like:

# My React Project

## How to run the project
npm start - starts development server
npm test - runs tests
npm run build - creates production build

## Code Style
- Use TypeScript
- Components go in src/components/
- Use functional components with hooks

## Important Notes
- Database is PostgreSQL
- API endpoints are in src/api/

The Best Way to Work with Claude

Explore → Plan → Code → Commit

  • "Read my files first" - Let Claude understand your code
  • "Don't write code yet" - Plan before coding
  • "Think about this" - Use analysis modes
  • "Break this down" - Make tasks smaller

What Can You Ask Claude?

Example conversations:

You: "Add a login button to my homepage"
Claude: "I'll read your homepage file first, then add a styled login button that matches your design."

You: "Why isn't my code working?"
Claude: "Let me check your files and find the bug. I see the issue on line 23..."

You: "Make this code cleaner"

Testing Made Easy

What is Test-Driven Development (TDD)?

  • Write the test first - Describe what you want your code to do
  • Watch it fail - The test should fail because you haven't written the code yet
  • Write minimal code - Just enough to make the test pass
  • Improve the code - Make it cleaner while keeping tests green

Red (Fail) → Green (Pass) → Refactor (Clean)

Essential Claude Code Commands

Master these commands to boost your productivity

Session Management

/clear
Start fresh conversation when switching topics
/compact
Compress long conversations while preserving context
/exit
End session and return to terminal
/resume
Resume previous conversation session

Information & Setup

/help
Show available commands and options
/init
Initialize Claude Code in new project
/model

/usage

Pro Tips for Better Results

  • Be specific: "Add a red button" vs "Add styling"
  • Show examples: Upload screenshots or paste code samples
  • Give context: "This is for a mobile app" vs "This is for a website"
  • Ask questions: "What's the best way to..." instead of guessing
  • Iterate: Refine your request if the first attempt isn't perfect

Troubleshooting Common Issues

Quick fixes for workshop scenarios

🚫 "Command not found: claude"
Solution: Restart terminal or run: export PATH=$PATH:/usr/local/bin
⚠️ "API Key not found"
Solution: Run /help then authenticate with claude auth login
🐌 "Claude is responding slowly"
Solution: Use /compact to compress conversation or /clear to start fresh
❌ "Claude can't find my files"
Solution: Make sure you're in the project directory and files exist. Use relative paths.
💸 "High token usage"
Solution: Check /usage, use /clear frequently, switch to sonnet with /model
🔄 "Lost my conversation context"
Solution: Try /resume to restore or recreate CLAUDE.md to give context

Common Mistakes to Avoid

  • Don't skip CLAUDE.md - It saves hours of explaining your project
  • Don't rush into coding - Let Claude explore your codebase first
  • Don't give vague instructions - Be specific about what you want
  • Don't ignore errors - Ask Claude to explain and fix them
  • Don't work alone - Use Claude as your coding partner
