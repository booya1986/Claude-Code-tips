What is Claude Code?

An AI-powered coding assistant in your terminal

  • Command-line interface (CLI) tool by Anthropic
  • Helps with coding tasks, debugging, and project management
  • Understands your entire codebase context
  • Can read, write, and modify files automatically
  • Integrates with your existing development workflow

What is CLAUDE.md?

Think of it as instructions for your AI assistant

  • A special file you create in your project folder
  • Contains project-specific information for Claude
  • Like giving Claude a "user manual" for your project
  • Helps Claude understand your project's unique setup

CLAUDE.md Example

Here's what a real CLAUDE.md file looks like:

# My React Project

## How to run the project
npm start - starts development server
npm test - runs tests
npm run build - creates production build

## Code Style
- Use TypeScript
- Components go in src/components/
- Use functional components with hooks

## Important Notes
- Database is PostgreSQL
- API endpoints are in src/api/

Why Use CLAUDE.md?

Benefits for beginners:

  • Saves time: Claude knows your project setup instantly
  • Prevents mistakes: Claude follows your coding standards
  • Better suggestions: Claude understands your tech stack
  • Team consistency: Everyone uses the same guidelines
  • Documentation: Your project is self-documenting

Getting Started - Step by Step

For complete beginners:

  • Step 1: Install Claude Code from Anthropic
  • Step 2: Open your terminal/command prompt
  • Step 3: Navigate to your project folder
  • Step 4: Create a CLAUDE.md file
  • Step 5: Start chatting with Claude about your code!

Testing Made Easy

What is Test-Driven Development (TDD)?

  • Write the test first - Describe what you want your code to do
  • Watch it fail - The test should fail because you haven't written the code yet
  • Write minimal code - Just enough to make the test pass
  • Improve the code - Make it cleaner while keeping tests green

Essential Commands for Beginners

Commands you'll use every day:

  • /clear - Start fresh conversation
  • /help - Get help with Claude Code
  • "Read this file" - Have Claude examine your code
  • "Explain this code" - Understand what code does
  • "Fix this bug" - Debug your code

Pro Tips for Better Results

  • Be specific: "Add a red button" vs "Add styling"
  • Show examples: Upload screenshots or paste code samples
  • Give context: "This is for a mobile app" vs "This is for a website"
  • Ask questions: "What's the best way to..." instead of guessing
  • Iterate: Refine your request if the first attempt isn't perfect

Common Mistakes to Avoid

  • Don't skip CLAUDE.md - It saves hours of explaining your project
  • Don't rush into coding - Let Claude explore your codebase first
  • Don't give vague instructions - Be specific about what you want
  • Don't ignore errors - Ask Claude to explain and fix them
  • Don't work alone - Use Claude as your coding partner
